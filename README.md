## Техническое задание

Составить программу на Delphi, которая должна выполнить следующие действия:

### 1. Загрузка данных из CSV в SQL
- Загрузить данные из CSV-файла в таблицу SQL (тестовый файл прилагается)
- **Формат файла:**
  - Кодировка: ANSI-1251
  - Разделитель столбцов: `;` (точка с запятой)
  - Разделитель строк: `#13#10` (CR+LF)
  - Поля: `Бренд`, `Артикул`, `Цена`, `Кол-во`
- Перед загрузкой создать целевую таблицу в базе `tempdb`

### 2. Удаление дубликатов
Удалить дубликаты **одним SQL-запросом**. Дубликатами считаются строки с одинаковыми полями `Бренд` и `Артикул`.

**Правила удаления:**
- Если у двух строк-дублей **разные цены** → удаляется строка с **меньшей ценой** (остаётся с большей)
- Если у двух дублей **одинаковая цена** → удаляется строка с **большим количеством** (остаётся с меньшим)
- Если равны **и цена, и количество** → удаляется любая строка (остаётся одна)

**Результат:** в таблице остаются только строки с уникальными сочетаниями `Бренд + Артикул`

### 3. Выгрузка результатов
Оставшиеся после удаления строки выгрузить в файл того же формата, что и исходный CSV.

### 4. Ограничения
- Использовать только компоненты из стандартного дистрибутива Delphi
- Разрешено использование модулей, поставляемых с MSSQL
- Версия Delphi: любая от 7 до современной

## Реализация

### 🛠 Технологии
- **Язык:** Delphi 7
- **СУБД:** Microsoft SQL Server (tempdb)
- **Конфигурация:** INI-файл

### ✨ Особенности реализации

#### 🔧 Конфигурация
- Строка подключения к БД вынесена в конфигурационный файл
- Файл `ProgramName.ini` создаётся автоматически при первом запуске в папке с программой
- При отсутствии файла создаётся с настройками по умолчанию

#### 📄 Парсинг CSV
- Собственный парсер (без использования сторонних библиотек)
- Учитывает особенности формата: разделитель `;`, кодировка ANSI-1251
- Корректная обработка пустых полей и пробелов

#### 🔍 Логика проверки существования таблицы, соответствия ее структуры и очистки содержимого
- При текущей логике нейминга таблицы и создании ее в tempdb с префиксом "##" вероятность сработывания стремится к 0
- Реализована, как демонстрация обработки таких ситуаций.

#### 🔤 Кодировка ANSI-1251 
- В текущей реализации строки передаются в SQL как Unicode, в рамках решаемой задачи это не принципиально, но фактически, на стороне SQL, такие строки будут некорректно интерпритироватся, если предполагается их хранение и обработка, разумеется, необходимо решить этот конфликт.

#### ⚡ Производительность
- **Пакетная вставка:** данные добавляются в БД пакетами (batch insert), что значительно повышает производительность по сравнению с построчным добавлением
- **Индексация:** для ускорения удаления дубликатов создаётся индекс по полям `Brand, Article`
- **Оптимизация удаления:** используется один SQL-запрос с оконной функцией `ROW_NUMBER()`

#### ⚠️ Примечания по безопасности
- Пакетная вставка потенциально уязвима к SQL-инъекциям.

- Для production-решения рекомендуется использовать параметризованные запросы

## Особенности проекта

### 📁 Структура
```
├── Unit1.pas          # Основной модуль программы
├── Unit1.dfm          # Форма
├── Test.dpr           # Файл проекта
├── Test.cfg           # Конфигурация проекта (настройки компилятора)
├── Test.res           # Ресурсы проекта (иконка, версионная информация)
├── Test.dof           # Опции проекта (параметры среды Delphi)
├── Test.exe           # Исполняемый файл
├── TestData.csv       # Тестовый файл с данными
```

### ⚙️ Настройки (INI-файл)
```ini
[Database]
ConnectionString=Provider=SQLOLEDB.1;Integrated Security=SSPI;Initial Catalog=tempdb;Data Source=(local);
Description=Строка подключения к SQL Server
```

### 📊 Алгоритм работы
1. Загрузка конфигурации из INI-файла
2. Подключение к `tempdb`
3. Создание/проверка структуры таблицы
4. Загрузка данных из CSV (пакетная вставка)
5. Создание индекса
6. Удаление дубликатов одним SQL-запросом
7. Выгрузка результата в CSV

## Требования

### Для запуска
- **Windows** (любая версия, поддерживающая Delphi 7)
- **Microsoft SQL Server** (любая редакция)
- Права на создание таблиц в `tempdb`(отозвать их проблематично и вредно, но возможно)

## Установка и запуск

1. Скомпилировать проект в Delphi 7
2. Запустить `Test.exe`
3. При первом запуске автоматически создастся `Test.ini`
4. При необходимости отредактировать строку подключения в INI-файле
5. Выбрать CSV-файл для обработки
6. Дождаться завершения
7. Результат сохранится в файл с суффиксом `_clean.csv`
